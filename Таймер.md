# Таймер

## Назначение

Модуль предоставляет таймеры и периодические тики для логики состояний и задержек в HSM.

## Идентификаторы из конфигурации

- Внутреннее имя: TimerModule
- Имя для HSM: Таймер
- Иконка: ⏱️

## Действия

### ТаймерЗапуск

Запускает таймер с заданными параметрами длительности и режима.

```
ТаймерЗапуск()
```

### ТаймерСтоп

Останавливает активный таймер без сброса накопленного состояния.

```
ТаймерСтоп()
```

### ТаймерСброс

Сбрасывает текущее значение таймера к начальному состоянию.

```
ТаймерСброс()
```

## События

### ТаймерВыполнен

Событие отправляется, когда таймер достигает конца отсчета.

### Тик1Секунда

Событие отправляется раз в секунду во время работы таймера.

### Тик

Событие отправляется на каждом шаге обновления таймера.

## Переменные

- ТекущееЗначениеТаймера

## Настройки

- **SILENT_TICK** — не логировать каждый тик таймера (значение: true)
- **SILENT_1S_TICK** — не логировать тики с интервалом 1 секунда (значение: true)

## Примечание

Рекомендуется использовать как единый источник задержек вместо ручного учета deltaTime в состояниях.
