# Триггер

## Назначение

Модуль отправляет триггерное событие объектам в радиусе, чтобы они могли отреагировать через свои HSM.

## Идентификаторы из конфигурации

- Внутреннее имя: TriggerModule
- Имя для HSM: Триггер
- Иконка: 🚪

## Действия

### ТриггерВРадиусе

Активирует триггер с указанным радиусом: модуль проверяет объекты вокруг центра и уведомляет попавшие в радиус цели.

```
ТриггерВРадиусе(radius, trigger_name)
```

**Аргументы:**

1. `radius` (число) — радиус в метрах
2. `trigger_name` (строка, необязательно) — имя триггера (если не указано, используется DEFAULT_NAME)

Особенности валидации:

- Если радиус отсутствует, равен `0`, нечисловой (`NaN`) или отрицательный, вызов игнорируется с логом ошибки.
- Если имя не передано, используется `settings.DEFAULT_NAME`.

## События

### ТриггерАктивированВРадиусе

Событие отправляется объекту, который попал в радиус активированного триггера.

Событие эмитится методом `reciveTrigger(...)` как при получении данных о триггере, так и при их сбросе (`reciveTrigger()` без аргументов).

## Переменные

- РасстояниеДоТриггера
  - Расстояние в мировых координатах от текущего объекта до источника триггера.
  - Возвращает `null`, если данных о последнем триггере нет.
- IDИсточникТриггера
  - Идентификатор объекта-источника триггера.
  - Возвращает `null`, если данных о последнем триггере нет.
- ИмяИсточникТриггера
  - Имя источника триггера.
  - Возвращает `null`, если данных о последнем триггере нет.

## Настройки (settings)

- DEFAULT_NAME
  - Значение по умолчанию для имени триггера, если `trigger_name` не передан.
  - Текущее значение в конфиге: `_unnamed_trigger_`.

## Примечание

Для корректной работы вычислений радиуса используйте мировые координаты (world position) центра триггера.
