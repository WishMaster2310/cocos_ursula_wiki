# МодульАктивации

## Назначение

Модуль управляет активацией и деактивацией объектов через команды HSM и отправляет события обратно в HSM.

## Идентификаторы из конфигурации

- Внутреннее имя: ActivationModule
- Имя для HSM: МодульАктивации
- Иконка: ⚡
- Радиус поиска: SEARCH_RADIUS (по умолчанию 10.0)

## Действия

### АктивироватьОбъект

```
АктивироватьОбъект(useInitialPosition)
```

**Аргументы:**

1. `useInitialPosition` (boolean) — использовать ли исходную позицию объекта

Поведение:

- Активирует все подходящие объекты из deactivatedRegistry в радиусе SEARCH_RADIUS.

### АктивироватьОбъектПоНазванию

```
АктивироватьОбъектПоНазванию(useInitialPosition, name)
```

**Аргументы:**

1. `useInitialPosition` (boolean) — использовать ли исходную позицию
2. `name` (строка) — имя объекта

Поведение:

- Активирует объекты в радиусе, где node.name равен name.

### АктивироватьОбъектПоID

```
АктивироватьОбъектПо ID(useInitialPosition, id)
```

**Аргументы:**

1. `useInitialPosition` (boolean) — использовать ли исходную позицию
2. `id` (строка) — идентификатор объекта (без символа @)

Поведение:

- Внутри id преобразуется в @id.
- Активирует объекты в радиусе, где ITEM_ID_KEY равен @id.

### ДеактивироватьЦель

```
ДеактивироватьЦель()
```

**Аргументы:**

Не требуются

Поведение:

- Берет текущую цель из TargetAnalyzer.
- Вызывает deactivateGameObject у целевого GameObject.

### ДеактивироватьСебя

```
ДеактивироватьСебя()
```

**Аргументы:**

Не требуются

Поведение:

- Вызывает deactivateGameObject у текущего объекта.

## События

### ОбъектАктивирован

Когда:

- На каждый успешно активированный объект.

Данные:

- data.target: Node

### ЦельДеактивирована

Когда:

- После успешного выполнения ДеактивироватьЦель.

Данные:

- не передаются

### Самодеактивация

Когда:

- После успешного выполнения ДеактивироватьСебя.

Данные:

- не передаются

### НетОбъектовДляАктивации

Когда:

- Если в deactivatedRegistry не нашлось ни одного кандидата в радиусе поиска.

Данные:

- не передаются

## Настройки

- **SEARCH_RADIUS** — радиус поиска объектов для активации (значение: 10.0 м)

## Фильтрация объектов при активации

Кандидат проходит фильтр, если:

1. node.isValid
2. node.active
3. node !== this.node
4. проходит дополнительный predicate (если задан)
5. расстояние до текущего объекта в мировых координатах <= SEARCH_RADIUS

Расстояние считается через Vec3.squaredDistance по world position.

## Зависимости

- TargetAnalyzer на том же узле (для ДеактивироватьЦель)
- GameObject на целевых узлах
- Заполненный deactivatedRegistry (обычно прокидывается из ObjectController)

## Важные фактические особенности текущей реализации

1. При ошибках (нет цели, нет TargetAnalyzer, нет GameObject) действие прерывается с логом ошибки, события не эмитятся.
2. Объекты с object_type `passable_environment` или `obstacle` пропускаются при активации и деактивации — для них выводится предупреждение.
