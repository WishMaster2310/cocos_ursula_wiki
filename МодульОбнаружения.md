# МодульОбнаружения

## Назначение

Модуль сканирует окружение в радиусе и сообщает HSM о найденных объектах, игроке, звуках и интерактивах.

## Идентификаторы из конфигурации

- Внутреннее имя: SensorModule
- Имя для HSM: МодульОбнаружения
- Иконка: 📡

## Действия

### ПоискИгрокаВРадиусе

Запускает сканирование на наличие игрока в радиусе сенсора.

```
ПоискИгрокаВРадиусе(radius)
```

**Аргументы:**

1. `radius` (число, необязательно) — радиус сканирования в метрах (если не указан, используется текущий радиус сенсора)

### ПоискОбъектаВРадиусеПоИмени

Ищет объект в радиусе по имени и обновляет данные о цели при нахождении.

```
ПоискОбъектаВРадиусеПоИмени(name, radius)
```

**Аргументы:**

1. `name` (строка) — имя объекта для поиска
2. `radius` (число) — радиус сканирования в метрах

**Пример:** `ПоискОбъектаВРадиусеПоИмени("Дверь", 10)` — искать объект с именем "Дверь" в радиусе 10 метров

### ПоискОбъектаВРадиусеПоИдентификатору

Ищет объект в радиусе по ID и обновляет данные о цели при нахождении.

```
ПоискОбъектаВРадиусеПоИдентификатору(id, radius)
```

**Аргументы:**

1. `id` (строка) — идентификатор объекта для поиска
2. `radius` (число) — радиус сканирования в метрах

**Пример:** `ПоискОбъектаВРадиусеПоИдентификатору("chest_01", 15)` — искать объект с ID "chest_01" в радиусе 15 метров

### ВзаимодействиеИгрокаСОбъектом

Проверяет факт взаимодействия игрока с объектом и отправляет соответствующее событие.
**_Важно_** Взаимодействием считает клик игроком по объекту, при этом, объект должен находится от игрока не дальше чем МодульВзаимодействия.ДистанцияВзаимодействия (по умолчанию 3 метра). Клик игроком по объекту на большем расстоянии будет проигнорирован

```
ВзаимодействиеИгрокаСОбъектом(identifier, radius)
```

**Аргументы:**

1. `identifier` (строка) — идентификатор или имя объекта для отслеживания взаимодействия
2. `radius` (число) — радиус в котором будет проверяться взаимодействие в метрах

**Пример:** `ВзаимодействиеИгрокаСОбъектом("Рычаг", 5)` — отслеживать взаимодействие игрока с объектом "Рычаг" в радиусе 5 метров

### УстановитьРадиусОбнаружения

Изменяет рабочий радиус сканирования сенсора.

```
УстановитьРадиусОбнаружения(radius)
```

**Аргументы:**

1. `radius` (число) — новый радиус сканирования в метрах

**Пример:** `УстановитьРадиусОбнаружения(25)` — установить радиус обнаружения 25 метров

### ОстановкаПоиска

Останавливает активный цикл сканирования.

```
ОстановкаПоиска()
```

### ПоискЗвуковВРадиусе

Ищет звуковые источники в заданном радиусе слышимости.

```
ПоискЗвуковВРадиусе(soundName, radius)
```

**Аргументы:**

1. `soundName` (строка) — имя звука для поиска
2. `radius` (число) — радиус поиска звуков в метрах

**Пример:** `ПоискЗвуковВРадиусе("gunshot", 50)` — искать звук "gunshot" в радиусе 50 метров

## События

### ИгрокОбнаружен

Событие отправляется, когда игрок найден в радиусе сенсора.

### ОбъектОбнаружен

Событие отправляется, когда найден требуемый объект.

### ИгрокНеОбнаружен

Событие отправляется, если игрок в радиусе не найден.

### ОбъектовНеОбнаружено

Событие отправляется, если подходящие объекты не найдены.

### ЗвукОбнаружен

Событие отправляется при обнаружении источника звука.

### ЗвукНеОбнаружен

Событие отправляется, если подходящий звук не найден.

### ВзаимодействиеИгрокаСОбъектом

Событие отправляется при подтвержденном взаимодействии игрока с текущим объектом.

### ВзаимодействиеИгрока

Событие отправляется как общий сигнал интеракции игрока.

## Переменные

- РадиусОбнаружения

## Настройки

- **DEFAULT_SENSOR_RADIUS** — радиус действия сенсора по умолчанию (значение: 15 м)
- **DEFAULT_SENSOR_SCAN_INTERVAL** — интервал сканирования (значение: 1.0 секунда)
- **MAX_SENSOR_RADIUS** — максимальный радиус сенсора (значение: 50 м)
- **SCAN_CACHE_LIFETIME** — время жизни кэша сканирования (значение: 5.0 секунд)

## Примечание

Модуль часто выступает источником событий для переключения состояний patrol/chase/engage.
