# МодульЭффектов

## Назначение

Модуль управляет применением различных эффектов к текущей цели объекта: лечение, урон, изменение скорости, сопротивления и пользовательские эффекты. Работает совместно с АнализаторЦели и EffectsComponent для определения цели и применения эффектов.

## Идентификаторы из конфигурации

- Внутреннее имя: EffectModule
- Имя для HSM: МодульЭффектов
- Иконка: ✨

## Входной API (actions)

### Лечение

Применяет к цели лечебный эффект, восстанавливающий здоровье.

```
Лечение(num_value, duration, is_periodic, is_stackable)
```

**Аргументы:**

1. `num_value` (число) — количество восстанавливаемого здоровья
2. `duration` (число) — длительность эффекта в секундах
3. `is_periodic` (boolean) — применять ли эффект периодически (каждую секунду)
4. `is_stackable` (boolean) — можно ли накладывать несколько таких эффектов

**Пример:** `Лечение(10, 5, false, false)` — восстановит 10 HP за 5 секунд (по 2 HP/сек)

### НанесениеУрона

Применяет к цели эффект урона здоровью.

```
НанесениеУрона(num_value, duration, is_periodic, is_stackable)
```

**Аргументы:**

1. `num_value` (число) — количество наносимого урона (автоматически становится отрицательным)
2. `duration` (число) — длительность эффекта в секундах
3. `is_periodic` (boolean) — применять ли эффект периодически (например, урон от горения)
4. `is_stackable` (boolean) — можно ли накладывать несколько таких эффектов

**Пример:** `НанесениеУрона(5, 10, true, false)` — наносит 5 урона каждую секунду в течение 10 секунд

### Ускорение

Применяет к цели эффект увеличения скорости передвижения.

```
Ускорение(percentage_value, duration, is_stackable)
```

**Аргументы:**

1. `percentage_value` (число) — процентное изменение скорости в формате 0..1 (0.2 = +20%, 1 = +100%)
2. `duration` (число) — длительность эффекта в секундах
3. `is_stackable` (boolean) — можно ли накладывать несколько таких эффектов

**Пример:** `Ускорение(0.5, 10, false)` — увеличит скорость на 50% на 10 секунд

### Замедление

Применяет к цели эффект снижения скорости передвижения.

```
Замедление(percentage_value, duration, is_stackable)
```

**Аргументы:**

1. `percentage_value` (число) — процентное изменение скорости в формате 0..1 (0.2 = -20%, автоматически становится отрицательным)
2. `duration` (число) — длительность эффекта в секундах
3. `is_stackable` (boolean) — можно ли накладывать несколько таких эффектов

**Пример:** `Замедление(0.3, 5, false)` — снизит скорость на 30% на 5 секунд

### ДобавитьСопротивление

Применяет к цели эффект сопротивления другому эффекту по имени.

```
ДобавитьСопротивление(effect_name, duration, resist_chance)
```

**Аргументы:**

1. `effect_name` (строка) — название эффекта, к которому добавляется сопротивление
2. `duration` (число) — длительность сопротивления в секундах
3. `resist_chance` (число) — шанс сопротивления от 0 до 1 (0.5 = 50% шанс заблокировать эффект)

**Пример:** `ДобавитьСопротивление("Горение", 10, 0.8)` — 80% шанс заблокировать эффект "Горение" на 10 секунд

### ДобавитьЭффект

Применяет к цели пользовательский эффект по имени из библиотеки эффектов.

```
ДобавитьЭффект(effect_name, duration)
```

**Аргументы:**

1. `effect_name` (строка) — название эффекта из библиотеки
2. `duration` (число) — длительность эффекта в секундах

**Пример:** `ДобавитьЭффект("Горение", 15)` — применяет эффект "Горение" на 15 секунд

### СнятьЭффектПоНазванию

Удаляет с цели конкретный эффект по его названию.

```
СнятьЭффектПоНазванию(effect_name)
```

**Аргументы:**

1. `effect_name` (строка) — название эффекта для удаления

**Пример:** `СнятьЭффектПоНазванию("Отравление")`

### СнятьВсеЭффекты

Удаляет с цели все активные эффекты (баффы, дебаффы, сопротивления).

```
СнятьВсеЭффекты()
```

### СнятьОтрицательныеЭффекты

Удаляет с цели только отрицательные эффекты (дебаффы).

```
СнятьОтрицательныеЭффекты()
```

### СнятьПоложительныеЭффекты

Удаляет с цели только положительные эффекты (баффы).

```
СнятьПоложительныеЭффекты()
```

### СнятьВсеСопротивления

Удаляет с цели все активные эффекты сопротивления.

```
СнятьВсеСопротивления()
```

## Выходной API (events)

### ЭффектДобавлен

Событие отправляется после успешного применения любого эффекта к цели.

**Данные события:**

- `effect` — объект примененного эффекта с его параметрами

### ПерезарядкаЗавершена

Событие отправляется когда завершается время перезарядки модуля и можно снова применять эффекты.

### ПерезарядкаНачата

Событие отправляется когда начинается перезарядка модуля после применения эффекта.

## Параметры (params)

- **ВремяДоПерезарядки** — текущее оставшееся время до готовности модуля (в секундах)

## Настройки (settings)

- **COOL_DOWN** — задержка между применением эффектов в секундах (по умолчанию: 3)

## Примечание

- Модуль требует наличия АнализаторЦели для определения цели применения эффектов
- Эффекты берутся из свойства `effects` (массив IEffect), которое должно быть заполнено из JSON или вручную
- Все действия по применению и снятию эффектов имеют кулдаун (перезарядку)
- Используйте названия действий, событий и параметров в HSM строго в кириллическом виде из `config.modules.EffectModule`
- Процентные значения указываются в формате 0..1, где 0.2 означает 20%, а 1 означает 100%
