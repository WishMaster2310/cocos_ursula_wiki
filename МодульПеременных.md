# МодульПеременных

## Назначение

Модуль управляет глобальными переменными состояния через StateController: устанавливает и сбрасывает значения, отслеживает изменения и отправляет события в HSM.

## Идентификаторы из конфигурации

- Внутреннее имя: StateModule
- Имя для HSM: МодульПеременных
- Иконка: 🎛️

## Действия

### УстановитьЗначение

Устанавливает значение переменной состояния.

```
УстановитьЗначение(var_name, value)
```

**Аргументы:**

1. `var_name` (строка) — имя переменной
2. `value` (любой тип) — новое значение

### СброситьЗначение

Сбрасывает переменную к начальному значению из конфигурации глобальных переменных.

```
СброситьЗначение(var_name)
```

**Аргументы:**

1. `var_name` (строка) — имя переменной

### СледитьЗаПеременной

Подписывает модуль на изменения указанной переменной. При изменении отправляется событие ЗначениеПеременнойИзменилось.

```
СледитьЗаПеременной(var_name)
```

**Аргументы:**

1. `var_name` (строка) — имя переменной

### ПрекратитьОтслеживание

Отписывает модуль от изменений конкретной переменной.

```
ПрекратитьОтслеживание(var_name)
```

**Аргументы:**

1. `var_name` (строка) — имя переменной

### ПрекратитьВсеОтслеживания

Снимает все активные подписки на переменные.

```
ПрекратитьВсеОтслеживания()
```

## События

### ЗначениеПеременнойИзменилось

Событие отправляется при изменении отслеживаемой переменной.

Данные события:

- data.name - имя переменной
- data.newValue - новое значение
- data.prevValue - предыдущее значение

## Переменные

Переменные читаются как значения переменных состояния по их именам.

Пример:

- getParam("quest_started") вернет текущее значение переменной quest_started.

## Настройки

- **MAX_VARIABLES** — максимальное количество переменных (значение: 42)

## Зависимости

- StateController должен быть инициализирован и передан в модуль (в GameObject при setupModules).

## Примечание

При деактивации и уничтожении модуль автоматически очищает все подписки на переменные.
